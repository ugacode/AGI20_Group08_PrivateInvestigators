Geolocation

The distinctive feature that characterizes the City Rat role is the ability for the player to walk outdoors in a geographically confined area in order to collect pieces of evidence. This is accomplished by displaying a custom map of the surrounding environment, with a virtual avatar positioned at the player's current position (detected via the device's GPS). By walking around, it is possible to get close enough to virtual objects – placed randomly on the map – to collect them and find out what evidence is hidden in them. For each one, the player will be presented a different challenge to be completed before unlocking a piece of evidence. Moving too fast in the environment will cause the player's "vision" to become blurry, thus making it more difficult to catch the spawned items.

Main technologies used for this feature were Unity (version 2019.4.10) and the Mapbox API, alongside Unity's post-processing stack.

Augmented Reality

One of the main features associated to the Genius role is the presence of AR in the game. It enables original and entertaining interaction modalities, which keep even a home-based player involved in the progress of the investigation. The user can select a piece of evidence from the inventory, place it in the surrounding environment, scale it to the desired dimensions and inspect its surface with a virtual UV lamp in search for potential clues. Unlocking the clues hidden on the different items will help bring the investigation forward.

Main technologies used for this feature were Unity (version 2019.4.10) and AR Core with AR Foundation, as the game was developed for Android platforms.

Gyroscope input and haptic feedback

With the intention of providing a fun, entertaining and unique challenge to the player taking the role of the City Rat, a minigame prototype was developed using input from the device's gyroscope and output using its vibration motors. The challenge is presented in the form of a vault lock, to be opened by turning the knob with the device until the right sequence of numbers is detected. By turning the device around, several audio "ticks", with different pitches, are played – one for each rotation step in the knob. A short vibration will help identify the step with the highest pitched "tick"; after holding the device still for a few seconds, a clue will be collected and stored in the inventory shared by all players.

Main technologies used for this feature were Unity (version 2019.4.10), UnityEngine's quaternion rotations and audio source pitch funtions for gyroscope input/output, along with Ruzrobert's custom vibration for Android devices in Unity.

Gesture Recognition

We considered for a while the idea of using gestures in AR to allow the Genius to manipulate any virtual object easly and effectively. We therefore explored the use of a gesture recognition software such as Manomotion with basic interaction, but we were slowed down by two issues; incompatibility between the version of Unity we were working on and the version supported by Manomotion; lack of any form of detailed documentation on how to implement basic functionalities, both in the official website and in third-party online forums. We decided to abandon the idea, to focus on other features.

Main technologies used for this feature were Unity (version 2020.1.11), Manomotion SDK (version 1.3) and AR Core with AR Foundation, as the game was developed for Android platforms.

3D modelling and animation

We have also explored 3D modelling and animation as side-branches to the project, with the aim of opening up the possibility of using our own assets instead of prefabs downloaded from the Internet. In particular, one main objective was modelling and animating a virtual avatar to be used in the virtual map for the City Rat role. The results of the exploration were arguably worse than the prefabs we were using since the start of the project, which we ended up keeping. We decided however to use our own custom model

Main technologies used for this feature were Blender (version 2.90.0) and Unity (version 2019.4.10).
